# Bug Log

This file tracks bugs encountered during development along with their solutions and prevention strategies.

## Format

```markdown
### YYYY-MM-DD - Brief Bug Description
- **Issue**: What went wrong
- **Root Cause**: Why it happened
- **Solution**: How it was fixed
- **Prevention**: How to avoid it in the future
```

## Guidelines

- Focus on recurring or instructive issues, not every minor bug
- Include enough context that someone unfamiliar can understand the problem
- Keep entries concise (1-3 lines per section)
- Add URLs to relevant documentation, monitoring dashboards, or tickets
- Older entries (6+ months) can be archived if no longer relevant
