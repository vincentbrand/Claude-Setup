# Work Log

This file tracks completed work, tickets, and issues for the project.

## Format

```markdown
### YYYY-MM-DD - TICKET-ID: Brief Description
- **Status**: Completed / In Progress / Blocked
- **Description**: 1-2 line summary of what was done
- **URL**: Link to ticket, PR, or documentation
- **Notes**: Any important context or follow-up items
```

## Guidelines

- Log completed work when finishing tickets or significant tasks
- Include ticket IDs for traceability
- Add URLs to tickets, pull requests, or related documentation
- Keep descriptions brief (1-2 lines)
- Archive older entries (3+ months) periodically to keep file manageable
- Status can be: Completed, In Progress, Blocked, or Cancelled
